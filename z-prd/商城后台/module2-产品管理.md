# 模块2：产品管理 PRD

## 一、需求背景与价值

### 1.1 需求背景
RebeccaShop 作为跨境电商 SaaS 平台，商品是商城最核心的数据资产。商家需要能够灵活地上架和管理商品信息，包括多语种内容、SKU 库存、SEO 优化、评论导入等。

### 1.2 用户价值
- 商家可以自主管理商品全生命周期（上架/下架/编辑/删除）
- 多语种支持使商品触达不同语种市场的消费者
- SKU 与仓库库存映射使库存管理更精准
- SEO 优化帮助商品获得更好的搜索引擎曝光

### 1.3 业务价值
- 商品数据是平台核心资产，完善的商品管理是商城运营基础
- 多语种 + 多币种天然支持跨境电商场景
- 评论导入功能降低商家迁移平台的成本

## 三、需求详情

### 3.1 商品列表页

**入口**：侧边栏 → 商品 → 商品列表

**页面结构**：
- 页面标题「商品列表(N件)」，N 为当前筛选条件下的商品总数
- 统计卡片：全部商品、已上架、草稿、已下架
- 搜索框：按商品标题/SPU编码搜索
- 筛选器：分类筛选（一级分类下拉）、状态筛选（上架/下架/草稿）、日期范围
- 批量操作栏：批量上架、批量下架、批量删除、导出
- 数据表格：包含以下列（支持自定义列显示/隐藏）
  - 复选框
  - 商品信息（主图缩略图 + SPU编码 + 标题）
  - 分类
  - 价格（售价/原价，美元显示）
  - 库存
  - 状态（上架/下架/草稿）
  - 创建时间
  - 操作（编辑、下架/上架、删除）

**交互逻辑**：
- 点击行进入编辑页面
- 点击"上架/下架"按钮即时切换状态，带二次确认
- 批量操作选中商品后出现批量操作栏
- 删除操作为软删除，带二次确认

### 3.2 创建/编辑商品页

**页面结构**：页面头部（面包屑 + 保存按钮）+ 语言Tab切换 + 左右两栏布局

#### 3.2.1 基本信息（左侧主体）

**多语言切换**：Tab 样式切换，默认显示当前店铺默认语言的版本，可切换录入其他语言版本。各语言版本独立存储。

每个语言版本包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品标题 | 文本输入 | 是 | 当前语言版本的标题 |
| 商品描述 | 富文本 | 否 | 当前语言版本的描述 |
| 产品分类 | 下拉选择 | 是 | 一级分类（品类来自中台预设的基础字典） |
| 属性1 | 文本输入 | 否 | 固定字段，单个值 |
| 属性2 | 文本输入 | 否 | 固定字段，单个值 |
| 属性3 | 文本输入 | 否 | 固定字段，单个值 |
| 属性4 | 文本输入 | 否 | 固定字段，单个值 |
| 属性5 | 文本输入 | 否 | 固定字段，单个值 |
| 产品标记 | 标签输入 | 否 | 一个输入框支持输入多个标签，回车添加，点击X删除 |
| SEO标题 | 文本输入 | 否 | 搜索引擎页面标题（`<title>`） |
| SEO描述 | 文本输入 | 否 | 搜索引擎页面描述（`<meta description>`） |
| URL网址名称 | 文本输入 | 否 | 商品落地页URL slug，自动从标题生成可手动修改 |

**属性1~属性5说明**：
- 5个固定字段，商家在后台自由填写
- 每个属性输入框只能填写一个值
- 不和中台品类做关联，纯自定义

**产品标记说明**：
- 类似 Shopify 的 Tags，一个输入框可输入多个标签
- 输入后回车添加标签，点击标签X删除
- 标签用于产品系列智能筛选和前端过滤

#### 3.2.2 商品媒体（左侧主体）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品主图 | 图片上传 | 是 | 支持多张轮播图，第一张为默认主图，支持拖拽排序 |
| 商品详情图 | 图片上传 | 否 | 商品详情页展示的图片/视频 |

#### 3.2.3 SKU规格（左侧主体）

| 功能 | 说明 |
|------|------|
| 自定义规格名 | 如：颜色、尺码、材质等，支持添加多个规格维度 |
| 规格值 | 每个规格名下添加多个规格值（如颜色：红/蓝/黑） |
| SKU表格 | 根据规格组合自动生成SKU行 |
| 购买基数 | 每个商品的最小购买数量，默认为1 |

**SKU表格列**：

| 列 | 说明 |
|----|------|
| 规格组合 | 自动拼接显示，如"红色 / XL" |
| 商品SKU编码 | 手动输入，唯一 |
| 库存SKU编码 | 手动输入，与库存系统货品SKU编码映射 |
| 原价(USD) | 以美元为基准的原价 |
| 售价(USD) | 以美元为基准的售价 |
| 库存数量 | 从库存系统同步显示（只读），或手动填写 |
| SKU图片 | 可选上传该SKU的配图 |
| 操作 | 删除SKU行 |

**SKU编码映射说明**：
- 商品SKU编码（商家自定义的商品编号）
- 库存SKU编码（库存系统中对应的货品编号）
- 两者通过手动填写建立映射关系
- 映射后可从库存系统拉取该SKU的库存数据

#### 3.2.4 价格与状态（右侧边栏）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品原价(USD) | 数字输入 | 否 | 划线价，以美元为基准 |
| 商品售价(USD) | 数字输入 | 是 | 以美元为基准，用户端根据汇率换算展示 |
| 商品状态 | 切换 | 是 | 上架/下架/草稿 |
| 征税设置 | 勾选 | 否 | 是否对该商品征税 |

**多币种价格说明**：
- 所有价格以美元（USD）为基准录入
- 用户端根据汇率模版（1USD = X币种）自动换算展示
- 汇率由管理员中台维护

#### 3.2.5 评论导入（右侧边栏）

| 功能 | 说明 |
|------|------|
| 下载模版 | 点击下载CSV评论模版文件 |
| 导入评论 | 上传CSV文件，批量导入到当前产品 |

**CSV模版格式**：

| 列名 | 必填 | 说明 |
|------|------|------|
| 用户昵称 | 是 | 评论者名称 |
| 评分 | 是 | 1-5星评分 |
| 评论标题 | 否 | 评论标题 |
| 评论内容 | 是 | 评论正文 |
| 评论时间 | 否 | 格式YYYY-MM-DD HH:mm:ss |

**导入逻辑**：
- 上传CSV后解析校验，显示预览表格
- 校验通过后确认导入
- 导入的评论绑定到当前产品ID
- 支持覆盖/追加模式选择

#### 3.2.6 SEO优化（右侧边栏 + 语言版本内）

每个语言版本独立设置SEO信息：

| 字段 | 说明 |
|------|------|
| 页面标题 | 搜索引擎结果显示的标题 |
| 页面描述 | 搜索引擎结果显示的描述摘要 |
| 网址名称 | 商品URL slug，如 `/products/red-dress` |

### 3.3 异常处理

| 场景 | 处理方式 |
|------|----------|
| 未填写必填字段 | 输入框标红，顶部Toast提示 |
| SPU编码重复 | 自动生成唯一编码，或提示用户修改 |
| SKU编码重复 | 行内标红提示 |
| 未录入任何语言版本 | 至少保存默认语言版本，弹窗提示 |
| CSV格式错误 | 解析失败行标红，提示用户修正后重新上传 |
| 网络超时 | 保存草稿，提示用户稍后重试 |

## 四、数据指标

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| 商品创建数 | 每日/每周/每月新建商品数 | 商品表创建时间 |
| 上架商品数 | 当前状态为上架的商品数 | 商品表状态字段 |
| SKU总数 | 所有商品的SKU行总数 | SKU表 |
| 多语种覆盖率 | 录入多语种的商品占比 | 商品语言版本表 |
| 评论导入成功率 | CSV导入成功数 / 总导入次数 | 评论导入日志 |
