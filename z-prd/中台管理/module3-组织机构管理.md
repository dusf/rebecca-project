# RebeccaShop 平台管理 - 组织机构管理 PRD

## 1. 需求背景与价值

### 1.1 背景
平台管理员需要对企业组织架构进行统一管理和维护，包括集团、公司、部门、小组等层级机构的增删改查、层级拖拽排序等操作。

### 1.2 价值
- **管理价值**：提供可视化的树形组织架构管理，支持层级拖拽，降低组织调整成本
- **业务价值**：组织架构是整个平台的基础数据，支撑店铺成员归属、权限分配等核心业务

## 3. 需求详情

### 3.1 组织列表页

#### 3.1.1 页面结构
- 入口：平台管理 → 账号管理 → 组织机构
- 页面结构：
  - 顶部操作栏：添加组织按钮 + 搜索框 + 类型筛选 + 状态筛选
  - 表格列：拖拽手柄 | 复选框 → 组织名称(树形展开) → 组织类型 → 上级组织 → 排序 → 状态 → 操作
  - 操作按钮：编辑、删除
- 组织类型：集团 > 公司 > 部门 > 小组（四级层级，直系父子关系）

#### 3.1.2 组织 ID 规则
- 系统自动生成唯一 ID，格式为类型前缀 + 递增序号
- 集团：G1, G2, G3...
- 公司：C1, C2, C3...
- 部门：D1, D2, D3...
- 小组：T1, T2, T3...
- **不设独立的"组织代码"字段**，以 ID 作为唯一标识

#### 3.1.3 排序规则
- 排序号由系统自动管理，**用户不可手动填写**
- 添加组织时，排序号自动取同级兄弟节点中最大排序号 + 1（首个为 1）
- 拖拽排序后，同级兄弟节点排序号自动重编号为 1, 2, 3... 连续数字
- 编辑组织时若修改了上级组织，自动在新父级下分配下一个序号，并重新整理旧父级的排序号

### 3.2 添加/编辑组织对话框

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 组织类型 | 下拉选择 | 是 | 集团/公司/部门/小组，编辑时若有子节点则不可修改 |
| 组织名称 | 文本输入 | 是 | 最多 50 字 |
| 上级组织 | 下拉选择 | 否 | 集团类型无上级，其他类型按直系父子层级规则过滤候选 |
| 状态 | 下拉选择 | 否 | 启用/停用，默认启用 |

- 注意：**排序字段不在表单中展示**，由系统自动分配

### 3.3 删除确认对话框
- 删除前展示被删组织及其所有下级组织名称
- 明确提示「此操作不可恢复」
- 二次确认后执行级联删除

### 3.4 拖拽排序
- 通过拖拽手柄（六点图标）进行拖拽
- 规则：仅允许拖拽到直系父级（如部门拖到公司下）或同级之间排序
- 拖拽到目标行上方/下方：同级排序重排
- 拖拽到目标行中间区域：变更父级
- 不允许跨层级拖拽（如小组直接拖到集团下），拖拽时显示无效状态
- 拖拽完成后排序号自动重编号为 1, 2, 3...
