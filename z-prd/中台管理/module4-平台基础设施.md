# RebeccaShop 平台管理 - 平台基础设施 PRD

## 1. 需求背景与价值

### 1.1 背景
platform/ 目录下各功能模块需要一个统一的前端代码组织规范和公共 UI 样式约定，确保代码可维护性和视觉一致性。

### 1.2 价值
- **开发效率**：统一的文件结构降低新模块的认知成本，公共样式减少重复代码
- **可维护性**：文件拆分后各模块职责清晰，对话框独立加载，修改范围可控

## 3. 需求详情

### 3.1 模块文件拆分规则

#### 3.1.1 标准目录结构
platform/ 下每个对象文件夹按需拆分为多个文件，**不允许把所有内容塞进单个 .html 文件**：

```
platform/<domain>/<object>/
├── <object>.html           # 主页面（极简骨架，仅页面结构 + 资源引用）
├── dialog-a.html            # 对话框 A 的 HTML 片段
├── dialog-b.html            # 对话框 B 的 HTML 片段
├── css/
│   └── <object>.css        # 页面专属样式
└── js/
    └── <object>.js         # 数据 + 渲染 + 交互逻辑
```

#### 3.1.2 各文件职责
| 文件 | 职责 | 规模要求 |
|------|------|----------|
| 主页面 .html | 页面骨架：顶部操作栏 + 表格容器 + 分页容器，引用 css/ 和 js/ 资源 | 精简，通常 < 100 行 |
| 对话框 .html | 单个对话框的 HTML 片段（含 overlay + dialog 结构） | 每个对话框一个文件 |
| css/ | 该模块所有样式（页面 + 对话框） | 按功能模块组织 |
| js/ | 数据定义、树形结构、表格渲染、对话框交互、拖拽、批量操作等 | 按功能模块组织 |

#### 3.1.3 对话框加载方式
- 对话框 HTML 通过 `fetch()` 异步加载，动态注入到 `document.body`
- 主页面 DOMContentLoaded 时统一加载所有对话框片段，加载完成后绑定事件
- 对话框必须包含右上角关闭图标（`.dialog-header` > `.dialog-title` + `.dialog-close`）

### 3.2 Select 下拉框统一样式

#### 3.2.1 样式规范
- 文件位置：`platform/common/css/platform.css`
- 适用范围：`select.form-input`（platform/ 下所有下拉框）
- 样式内容：
  - 隐藏原生浏览器箭头（`appearance: none`）
  - 使用自定义 SVG 下拉箭头图标（灰色 `#90A3B8`，14x14px）
  - 图标定位 `right 12px center`，右侧留 12px 间距
  - 右侧总 padding 为 34px（14px 内边距 + 图标空间 + 安全间距）
  - 悬停和聚焦态沿用原有 form-input 交互样式
